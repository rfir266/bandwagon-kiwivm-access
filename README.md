## 方法一：从官网后台跳转进入（最常用，推荐）

这是大多数人用的方式，不需要记额外的密码，直接靠搬瓦工账号就能进去。

步骤如下：

1. 打开搬瓦工官网，用注册账号登录，进入 Client Area
2. 点击顶部菜单的 **Services** → **My Services**
3. 页面会列出你名下所有 VPS，找到你想管理的那一台
4. 点击该 VPS 条目右边的 **KiwiVM Control Panel** 按钮，或者点进去之后在左侧操作菜单找到该按钮
5. 直接跳转进入 KiwiVM 面板，无需额外输入密码

这种方式的好处是一步到位——账号验证已经在官网那层做完了，KiwiVM 会自动识别。多台 VPS 的用户在这个列表里切换就行，每台 VPS 有自己独立的 KiwiVM 面板。

## 方法二：直接访问 KiwiVM 地址（应急或需要单独密码时用）

KiwiVM 有自己的独立登录页，地址是 `kiwivm.64clouds.com`。打开之后是一个登录界面，需要输入：

- **用户名**：你这台 VPS 的 IP 地址
- **密码**：KiwiVM 面板的独立密码（注意：这个密码不是搬瓦工账号密码，也不是 SSH 登录密码，是另外一个独立的密码）

这里有个问题：KiwiVM 面板的独立密码，默认状态下很多用户其实没有设置过，也不知道是什么。

**没设置过怎么办？** 进去之后在左侧菜单找到 **KiwiVM password modification**，设置一个新密码就行了。设完之后这个页面就能用了。

这个方式主要用在两种场景：一是你把 VPS 转借给别人用，想给对方一个单独的面板入口而不想暴露主账号；二是出于习惯或某种原因想直接走面板地址。平时用方法一就够了。

## 忘记 KiwiVM 密码怎么办

分两种情况说：

**情况一：忘记的是 KiwiVM 独立密码**

在 KiwiVM 登录页面底部，有一个 "Reset KiwiVM password" 链接，点进去输入你的 VPS IP 地址，然后点 **Initiate password reset**。系统会把重置链接发到你的注册邮箱。注意链接有效期只有 20 分钟，收到邮件后尽快点击，按提示设置新密码即可。

**情况二：忘记的是 SSH/root 密码（连不上服务器）**

这个跟 KiwiVM 密码是两回事。root 密码是用来 SSH 登录 VPS 的密码，跟 KiwiVM 面板密码完全独立。

处理方法：用方法一从官网跳转进 KiwiVM，进去之后在左侧菜单找 **Root password modification**，点 **Generate and set new root password** 按钮，系统会生成一个新密码并显示在页面上，同时发到注册邮箱。

注意一点：这个密码页面关掉就看不到了。显示出来的时候马上复制保存，别等到发现邮件进垃圾箱才去找。

## 登录后能做什么：KiwiVM 面板功能速览

进去之后左侧是功能菜单，主要分几个区：

**主要操作（Main Controls）**
- 开机、关机、重启 VPS——对应的按钮是 Start / Stop / Restart
- 查看 VPS 当前状态：IP 地址、SSH 端口号、内存/硬盘/流量使用情况
- 截图：实时查看 VPS 当前的 VNC 界面截图，用于排查黑屏问题

**系统管理**
- **Install new OS**：重装操作系统，支持 Ubuntu、Debian、CentOS、Rocky Linux 等，重装前需要先关机
- **Root password modification**：重置 SSH root 密码
- **KiwiVM password modification**：修改 KiwiVM 面板独立密码

**迁移与网络**
- **Migrate to another DC**：一键迁移机房，CN2 GIA-E 套餐可切换 12 个以上机房，操作免费，次数不限
- **Separate Private Network**：添加私有 IP，用于同机房 VPS 之间组内网，内网流量不计入总流量

**高级操作**
- **Root Shell**（Basic / Advanced / Interactive）：提供在线终端，浏览器里就能直接敲命令，不用打开 SSH 客户端
- **Two-factor authentication**：开启谷歌二步验证，防止账号被盗后面板被直接操作
- **Snapshots**：快照功能，最多保存 2 个快照，免费
- **Audit Log**：查看面板操作日志，方便追溯谁在什么时候做了什么操作

每台 VPS 的 SSH 端口不是默认的 22，而是搬瓦工单独分配的五位数端口——在 Main Controls 页面里可以看到，初次连接之前一定要记下来。

## 关于 KiwiVM 安全性：要不要开二步验证

说实话，如果你平时只用方法一（从官网后台跳转进入），KiwiVM 面板自身的安全性依赖的是搬瓦工账号的安全性。账号密码够强、不乱登陌生设备，基本没问题。

但如果你在用直接访问 KiwiVM 地址的方式，或者 VPS 里跑的东西比较重要，建议开一下 **Two-factor authentication**。

开启步骤：
1. 进入 KiwiVM 面板，左侧菜单点 **Two-factor authentication**
2. 点 Continue 进入下一步
3. 手机下载 Google Authenticator（国内安卓机不好用谷歌服务的，可以用 Authy 或 Microsoft Authenticator，功能一样）
4. 扫描页面上的二维码，或者手动输入 account key 添加账户
5. 完成绑定后，每次直接访问 KiwiVM 地址登录都需要额外输入验证码

开启之后有一点要注意：**一定要把 Backup Key 保存好**。手机丢了或者换手机了，只有这个 Key 才能恢复访问。这个 Key 只在设置时显示一次，没保存就只能联系搬瓦工客服了。

## 常见问题

**Q：搬瓦工 KiwiVM 面板是什么，跟 SSH 连接有什么区别？**

A：KiwiVM 是搬瓦工官方提供的 VPS 管理面板，管的是服务器硬件层面的操作，比如开关机、重装系统、查看 IP 和端口、迁移机房。SSH 是用来远程操作服务器系统内部的，两个东西是分层的：先通过 KiwiVM 查到 IP 和端口，再用 SSH 客户端连进去操作。

**Q：KiwiVM 登录界面出现 "Session Expired" 怎么处理？**

A：这个通常是 Cookie 过期或者浏览器缓存问题。清一下浏览器缓存，或者换一个浏览器重新从官网后台点 KiwiVM Control Panel 按钮进去，一般就好了。

**Q：搬瓦工有几套密码，分别是什么？**

A：三套独立的密码：① 搬瓦工官网账号密码（登录 Client Area 用的）；② KiwiVM 面板独立密码（直接访问 kiwivm.64clouds.com 时用到，不设的话不需要）；③ VPS 的 root 密码（SSH 连接 VPS 用的）。三个密码完全独立，改一个不影响另外两个。

**Q：进了 KiwiVM 显示 VPS 已停止（Stopped），是什么情况？**

A：有几种可能：流量用完了被暂停（Main Controls 里看 Bandwidth usage，超了会显示提示）；或者 VPS 因为某个原因宕机。在面板里点 Start 重新启动，看看能否恢复。流量超量的话需要等到下个月重置，或者购买额外流量包。

**Q：KiwiVM 面板里的迁移机房功能是免费的吗？**

A：免费，次数不限。切换一次会把迁移过程中的流量算入总流量消耗，正常情况下这部分消耗很少。注意如果 IP 被封，迁移机房不能解决封 IP 的问题，需要先付费换 IP 再迁移。
